# 프로젝트명: 트러스트 라인 (Trust-Line)
## 지인 간 채무 관리 및 법적 증빙 지원 서비스 (MVP)

### 1. 프로젝트 개요
현대 사회에서 지인 간 금전 거래 시 발생하는 심리적 부담을 줄이고, '을이 된 채권자'를 보호하기 위한 플랫폼입니다. 명확한 전자 계약과 시스템 주도의 상환 관리를 통해 신뢰를 유지하고, 불이행 시 법적 대응을 돕습니다.

### 2. 핵심 원칙 (MVP)
- **단순함:** 일시 상환을 원칙으로 하며, 복잡한 분할 상환은 제외함.
- **실효성:** 본인 인증 기반 전자 서명으로 법적 증거 능력 확보.
- **자동화:** 시스템(가상계좌 웹훅)이 입금을 감지하고 상태를 자동 업데이트.

### 3. 데이터 모델 (Entity 설계 요구사항)
1. **Member (사용자)**
    - 이름, 이메일, 자체 신뢰 점수 (기본 100점)
2. **Contract (계약)**
    - 채권자/채무자 (Member와 연관 관계)
    - 대여 금액, 상환 예정일, 주문번호(orderId - UUID)
    - 상태: PENDING(대기), PROCEEDING(진행), COMPLETED(완료), FAILED(불이행)
3. **RepaymentSchedule (상환 일정)**
    - 계약당 1개 생성 (추후 확장성 고려)
    - 상태: PLANNED(예정), DONE(완료), OVERDUE(연체)
4. **TrustHistory (신뢰 이력)**
    - 점수 변동 사유, 변동 수치, 기록 일시

### 4. 핵심 비즈니스 로직
- **상환 성공:** 신뢰 등급 유지 및 '클린 상환 인증서' 데이터 확정.
- **상환 실패:** 상환일 경과 시 즉시 FAILED 처리, 신뢰 점수 -10점, 서비스 이용 제한 및 '소송용 증거 PDF' 생성 데이터 확정.
